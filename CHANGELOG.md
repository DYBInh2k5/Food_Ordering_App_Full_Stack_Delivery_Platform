# Changelog

All notable changes to this project are documented in this file.

The format is based on Keep a Changelog.
Versioning follows Semantic Versioning.

## [Unreleased]
### Added
- Split CI workflows by backend and frontend scope.
- Auto GitHub release workflow triggered by version tags.
- CodeQL security scanning workflow.
- Dependabot configuration for npm and GitHub Actions.
- Repository labels configuration for project triage.
- GitHub issue templates for bug reports and feature requests.
- Pull request template with quality and risk checklist.
- ADR documentation folder with template.

### Changed
- Documentation structure improved for long-term maintenance.

## [1.0.1-beta] - 2026-03-25
### Added
- Backend and frontend CI workflows with scoped path triggers.
- Automated release publishing workflow on tag push.
- Weekly dependency update checks with Dependabot.
- Security code analysis with CodeQL.
- GitHub project setup guide for operations governance.

## [1.0.0-beta] - 2026-03-25
### Added
- Full-stack food delivery platform with role-based flows.
- Customer pages for auth, browsing, cart, checkout, orders, profile, reviews.
- Restaurant dashboard for order and menu management.
- Driver dashboard for assignment, tracking, and completion.
- Review and rating system for restaurants and drivers.
- Comprehensive deployment and setup documentation.

### Technical
- Backend: Express, MongoDB, Mongoose, JWT, bcrypt.
- Frontend: React, Redux Toolkit, React Router, Tailwind CSS, Axios.
- API: 50+ endpoints with layered architecture.
